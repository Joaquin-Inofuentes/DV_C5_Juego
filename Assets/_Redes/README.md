# Redes — Parcial 2 (Photon Fusion 2, Host) — Top-Down Shooter

**Scaffold only.** This delivers the full structure + relationships (no internal logic).
Every script lives under the `Redes.*` namespace and inside `Assets/_Redes`.
Another agent implements the bodies marked `// TODO (other agent)`.

## How to generate the project (Unity Editor)
Run the three menu items **in order**:

1. `Tools > Redes > 1. Create Scene` → creates `Assets/_Redes/Scenes/RedesGame.unity`
   (camera top-down, ground, legacy-Text UI, NetworkRunner, GameManager, Controllers).
2. `Tools > Redes > 2. Create Prefabs` → creates `Assets/_Redes/Prefabs/Player.prefab`
   (capsule + all player systems) and `Bullet.prefab` (sphere + Projectile).
3. `Tools > Redes > 3. Link & Assign All` → wires every serialized reference between
   network ↔ controllers ↔ views and drops the prefabs into their fields.

After step 3: register `Player` and `Bullet` in Fusion's **NetworkProjectConfig**
(Fusion > Network Project Config) so the runner can spawn them.

## Requirement mapping
- **Network Runner** → `NetworkRunner` on the `NetworkRunner` GameObject (+ `HostNetworkService`).
- **Network Object** → `NetworkObject` on Player, Bullet and GameManager.
- **Animation control** → `PlayerAnimationController` (drives Animator in `Render()`).
- **Host architecture** → `HostNetworkService` starts `GameMode.Host`; only server spawns.
- **No start until 2 players** → `GameConstants.MIN_PLAYERS_TO_START = 2`,
  `OnEnoughPlayersToStart` fired from `RefreshPlayerCount()`.
- **Win/Lose notified to all** → `MatchNetworkController` RPC → each client resolves
  result → `MatchController` → `ResultView.ShowResult` fires an `Action` ("con action").
- **No desync** → `[Networked]` state (health, ammo) + simulation in `FixedUpdateNetwork`
  + single RPC resolved identically on every client.
- **Extra mechanic (≠ Move/Shoot/Jump)** → `AmmoSystem` (limited magazine + reload).

## SOLID / MVC
- **Model**: `GameStateModel`, `PlayerModel` (pure data + events).
- **View**: `LobbyView`, `GameHudView`, `ResultView` (legacy Text only, no logic).
- **Controller**: `GameFlowController`, `MatchController`, `PlayerController`.
- **Interfaces (DIP/ISP/SRP)**: `INetworkService`, `ISessionListHandler`, `IDamageable`.

## Console logs (flags) you will see
All logs go through `RedesLog` with colored flags: `[REDES][NET/LOBBY/MATCH/PLAYER/COMBAT/AMMO]`.

Required messages (already placed in the correct methods):
- `Inicio el juego` — `HostNetworkService.StartAsHost`
- `Se encontraron 0 salas` / `Se encontraron X salas` — `OnSessionListUpdated`
- `Se creo una sala llamada X` / `Se esta esperando al otro jugador` — `RoomSessionHandler` (create path)
- `Se unio a la sala de X nombre y X datos` — `RoomSessionHandler` (join path)
- `se inicio el juego por q se tienen 2 jugadores` — `RefreshPlayerCount`
- `Inicio el jugador A` / `Inicio el Jugador B` — `NetworkPlayer.Spawned`
- `El jugador B disparo` — `PlayerShooting.Fire`
- `El jugador B recibio el impacto` — `PlayerHealth.TakeDamage`
- `El jugador A Perdio` — `PlayerHealth.TakeDamage` (on death)
- `...recibio la notitifacion de q gano con action` / `...de q perdio con action` — `ResultView.ShowResult`
